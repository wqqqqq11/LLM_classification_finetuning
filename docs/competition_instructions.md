Kaggle LLM 用户偏好预测竞赛
一、赛事整体概况
1. 比赛核心任务
本次竞赛要求选手搭建机器学习模型，预测用户在两款匿名大模型（LLM）作答同一条用户提示词的一对一 PK 场景中，最终会偏好哪一个回复；训练数据来源于 Chatbot Arena 平台真实人机对话与用户投票选择数据集。
2. 赛事科研与落地价值
该任务对应 ** 基于人类反馈的强化学习（RLHF）** 里的偏好模型 / 奖励模型研发方向；现有直接用大模型预判用户偏好的方案存在三大固有偏差：优先选择靠前展示回答的位置偏见、偏爱长篇回复的冗长偏见、模型自我吹捧的自抬偏见，选手建模可弥补上述缺陷。最终成果能够优化大模型适配人类偏好能力，迭代出更贴合用户使用习惯的对话 AI 系统。
二、Kaggle 参赛标准流程
加入赛事：阅读赛事说明、勾选同意比赛规则，解锁数据集下载权限；
模型开发：新建 Kaggle Notebook，导入赛事数据、完成模型搭建，生成预测结果文件；
提交参赛：将调试完毕的 Notebook 提交至赛事系统；
查看排名：通过排行榜查看自身模型与其他参赛选手的得分位次；
迭代提分：在赛事讨论区参考其他选手分享的教程、解题思路优化模型。
三、模型评测规则 & 提交文件规范
1. 评价指标
采用 ** 对数损失（log loss）** 作为打分依据，计算时参数设置eps=auto，通过模型预测概率和真实标签的误差衡量模型效果。
2. 结果提交文件格式
文件命名强制为：submission.csv；
文件首行必须为表头，字段：id,winner_model_a,winner_model_b,winner_tie；
字段含义：每条测试样本 ID 对应输出三个概率值，分别是用户选择 A 模型、选择 B 模型、双方平局的概率，三类概率总和为 1；
样例：
csv
id,winner_model_a,winner_model_b,winner_tie
136060,0.33,0.33,0.33
211333,0.33,0.33,0.33
1233961,0.33,0.33,0.33


数据集说明
一、数据集来源与业务背景
数据集全部取自 ChatBot Arena 平台的真实用户交互数据：单条交互中，评测用户输入一条或多条提示词，交由两个不同大语言模型分别生成回复，之后用户从两份回复里选出更满意的答案；比赛任务就是基于该数据建模，预测评测人的偏好结果，预估每组「提示词 + 两份模型回复」组合被选为优胜方的概率。
赛事属性补充：本赛事为代码赛，平台计分环节会用完整测试集替换示例测试数据；训练集约55000 行，测试集约25000 行。
二、参赛文件清单与字段释义
1. train.csv（训练集）
表格
字段名	字段说明
id	单条样本唯一编号
model_a、model_b	对应 A、B 两个大模型的标识；仅训练集包含该字段，测试集无此字段
prompt	输入给两个模型的用户提示词（两份模型共用同一个 prompt）
response_a、response_b	分别是模型 A、模型 B 针对同一提示词生成的回复文本
winner_model_a / winner_model_b / winner_tie	二元标签字段，代表评测人真实选择结果，是模型需要拟合的真实标签（目标列）
2. test.csv（测试集）
仅包含 id、prompt、response_a、response_b 四个字段，不含模型名称字段、不含获胜结果标签字段，需要选手基于该文件预测三类获胜概率。
3. sample_submission.csv（提交样例文件）
提供标准提交格式模板，字段：
id：测试样本编号
winner_model_a、winner_model_b、winner_tie：选手需要输出的三类预测概率（选 A、选 B、平局）
三、补充备注
测试阶段官方会替换完整真实测试数据，不能依赖示例测试数据分布；
